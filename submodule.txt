Подмодуль git — это запись в хост-репозитории git, которая указывает на конкретный коммит в другом внешнем репозитории. Подмодули очень статичны: с их помощью можно отслеживать определенные коммиты, но не ветки или ссылки git. Кроме того, они не обновляются автоматически при обновлении хост-репозитория. Когда вы добавляете подмодуль в репозиторий, создается новый файл .gitmodules, в котором содержатся метаданные о сопоставлении URL-адреса проекта подмодуля с локальным каталогом. Если хост-репозиторий содержит несколько подмодулей, в файл .gitmodules будет включена отдельная запись для каждого подмодуля.


git submodule - это механизм в системе контроля версий Git, который позволяет включать один репозиторий Git внутри другого репозитория Git. Такие вложенные репозитории называются подмодулями. Основная цель подмодулей - управление зависимостями и включение в проект сторонних компонентов или библиотек.

Основные понятия, связанные с подмодулями Git:

Главный репозиторий (суперпроект): Это верхний уровень вашего проекта, который включает в себя подмодули. Главный репозиторий содержит ссылки (или указатели) на подмодули, указывая на определенные коммиты в их репозиториях.

Подмодуль: Это вложенный репозиторий, который находится внутри главного репозитория. Он может представлять собой отдельный проект, библиотеку или компонент, который используется в вашем проекте.

Преимущества использования подмодулей:

Управление зависимостями: Подмодули позволяют вам включать сторонние компоненты или библиотеки в ваш проект без необходимости копировать их код внутрь вашего репозитория.
Версионирование: Вы можете указывать конкретные версии подмодулей, что облегчает совместимость с определенными версиями зависимых компонентов.
Разделение проектов: Разработка происходит независимо в главном репозитории и подмодулях, что облегчает сопровождение и обновление зависимостей.
